# API 二级规范模板

## 使用说明

新增 API 规范时：
1. 复制本文件内容
2. 重命名为 `<API_NAME>_<功能>_API.md`
3. 填入实际 API 信息
4. 更新 [API_DESIGN.md](API_DESIGN.md) 索引

---

## 模板结构

```markdown
# <API名称> <功能> 规范

## 概述
[功能说明、使用场景、版本]

## API 信息
| 项目 | 值 |
|------|-----|
| 端点 | |
| 认证方式 | |
| 请求限制 | |
| 参考文档 | |

## 请求格式
- 请求头
- 必填参数（表格）
- 可选参数（表格）

## 响应格式
- 成功响应结构
- 错误响应结构
- Dart 解析代码

## 关键格式要求
- ✅ 正确示例
- ❌ 错误示例

## 错误处理
- 错误码对照表
- 重试策略

## 测试案例
- 场景、输入、预期输出、实际结果

## 修改记录
| 日期 | 修改内容 |
```
