# 畅记APP - 本地构建指南

## ⚠️ 重要说明

由于当前环境的网络限制和权限配置，Flutter SDK的Dart组件无法完成初始化下载。因此，你需要在**本地完整环境**中执行以下步骤来构建APP。

---

## 📋 环境准备

### 1. 系统要求

- **操作系统**: Windows 10/11、macOS 12+ 或 Linux Ubuntu 20.04+
- **内存**: 至少 8GB RAM（推荐 16GB）
- **磁盘空间**: 至少 10GB 可用空间
- **网络**: 稳定的互联网连接（需要下载约 2-3GB 文件）

### 2. 必需软件

#### Windows

1. **Git for Windows**
   - 下载地址: https://git-scm.com/download/win
   - 安装时选择 "Use Git from the Windows Command Prompt"

2. **Android Studio**
   - 下载地址: https://developer.android.com/studio
   - 安装时勾选:
     - Android SDK
     - Android SDK Platform
     - Android Virtual Device (可选)

3. **Java JDK 17**
   - 下载地址: https://adoptium.net/temurin/releases/?version=17
   - 安装后配置环境变量 `JAVA_HOME`

#### macOS

1. **Homebrew** (包管理器)
   ```bash
   /bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
   ```

2. **Android Studio**
   - 下载地址: https://developer.android.com/studio

3. **Java JDK 17**
   ```bash
   brew install --cask temurin17
   ```

## 🛠️ 安装 Flutter SDK

### Windows

```powershell
# 1. 创建安装目录
mkdir C:\flutter-sdk

# 2. 克隆 Flutter SDK (使用国内镜像加速)
git clone https://mirrors.tuna.tsinghua.edu.cn/git/flutter-sdk.git -b stable C:\flutter-sdk
# 或者使用官方源
git clone https://github.com/flutter/flutter.git -b stable C:\flutter-sdk

# 3. 添加环境变量
[Environment]::SetEnvironmentVariable("Path", $env:Path + ";C:\flutter-sdk\bin", "User")
[Environment]::SetEnvironmentVariable("FLUTTER_STORAGE_BASE_URL", "https://mirrors.tuna.tsinghua.edu.cn/flutter", "User")
[Environment]::SetEnvironmentVariable("PUB_HOSTED_URL", "https://mirrors.tuna.tsinghua.edu.cn/dart-pub", "User")

# 4. 重启终端，验证安装
flutter doctor
```

### macOS

```bash
# 1. 克隆 Flutter SDK
git clone https://github.com/flutter/flutter.git -b stable ~/flutter

# 2. 添加环境变量
echo 'export PATH="$PATH:$HOME/flutter/bin"' >> ~/.zshrc
echo 'export FLUTTER_STORAGE_BASE_URL=https://mirrors.tuna.tsinghua.edu.cn/flutter' >> ~/.zshrc
echo 'export PUB_HOSTED_URL=https://mirrors.tuna.tsinghua.edu.cn/dart-pub' >> ~/.zshrc
source ~/.zshrc

# 3. 验证安装
flutter doctor
```

### Linux

```bash
# 1. 克隆 Flutter SDK
git clone https://github.com/flutter/flutter.git -b stable ~/flutter

# 2. 添加环境变量
echo 'export PATH="$PATH:$HOME/flutter/bin"' >> ~/.bashrc
source ~/.bashrc

# 3. 验证安装
flutter doctor
```

## ⚙️ 配置 Android SDK

### 1. 打开 Android Studio

1. 启动 Android Studio
2. 点击 "More Actions" → "SDK Manager"
3. 在 "SDK Platforms" 标签页:
   - 勾选 Android API 34 (或最新版本)
   - 勾选 Android API 21 (最低支持版本)
4. 在 "SDK Tools" 标签页:
   - 勾选 Android SDK Build-Tools
   - 勾选 Android SDK Command-line Tools
   - 勾选 Android Emulator (可选)
   - 勾选 Android SDK Platform-Tools
5. 点击 "Apply" 安装

### 2. 配置环境变量

#### Windows

```powershell
# 设置 ANDROID_HOME
[Environment]::SetEnvironmentVariable("ANDROID_HOME", "$env:LOCALAPPDATA\Android\Sdk", "User")

# 添加 platform-tools 到 Path
[Environment]::SetEnvironmentVariable("Path", $env:Path + ";%ANDROID_HOME%\platform-tools", "User")
```

#### macOS/Linux

```bash
# 添加到 ~/.zshrc 或 ~/.bashrc
echo 'export ANDROID_HOME=$HOME/Library/Android/sdk' >> ~/.zshrc
echo 'export PATH=$PATH:$ANDROID_HOME/platform-tools' >> ~/.zshrc
source ~/.zshrc
```

## 📦 构建畅记APP

### 1. 准备项目

```bash
# 进入项目目录
cd changji_app

# 验证 Flutter 环境
flutter doctor

# 获取依赖（使用国内镜像加速）
flutter pub get
```

### 2. 生成必要代码

```bash
# 生成 Drift 数据库代码
flutter pub run build_runner build --delete-conflicting-outputs

# 如果失败，尝试
flutter pub run build_runner build
```

### 3. 构建 APK

#### 调试版本（用于测试）

```bash
# 构建调试版 APK
flutter build apk --debug

# 输出路径: build/app/outputs/flutter-apk/app-debug.apk
```

#### 发布版本（用于分发）

```bash
# 构建发布版 APK
flutter build apk --release

# 输出路径: build/app/outputs/flutter-apk/app-release.apk
```

#### 分架构构建（减小包体积）

```bash
# 按 CPU 架构分包
flutter build apk --release --split-per-abi

# 输出:
# - app-arm64-v8a-release.apk
# - app-armeabi-v7a-release.apk
# - app-x86_64-release.apk
```

### 4. 构建 App Bundle (Google Play)

```bash
flutter build appbundle --release

# 输出路径: build/app/outputs/bundle/release/app-release.aab
```

## 🔧 常见问题解决

### 1. Flutter 命令无法识别

**症状**: `flutter` 命令无法找到

**解决**:
```bash
# 检查环境变量
echo $PATH  # macOS/Linux
$env:Path   # PowerShell

# 手动添加路径
# Windows: 系统属性 → 环境变量 → Path → 编辑 → 新建 → C:\flutter-sdk\bin
# macOS/Linux: export PATH="$PATH:/path/to/flutter/bin"
```

### 2. Dart SDK 下载失败

**症状**: `flutter doctor` 卡在 "Downloading Dart SDK"

**解决**:
```bash
# 使用国内镜像
export FLUTTER_STORAGE_BASE_URL=https://mirrors.tuna.tsinghua.edu.cn/flutter
export PUB_HOSTED_URL=https://mirrors.tuna.tsinghua.edu.cn/dart-pub

# 手动下载 Dart SDK
# 1. 查看 Flutter 版本对应的 Dart 版本
flutter --version

# 2. 从 https://dart.dev/get-dart/archive 下载对应版本
# 3. 解压到 flutter/bin/cache/dart-sdk/
```

### 3. Gradle 构建失败

**症状**: 构建时 Gradle 报错

**解决**:
```bash
# 清理构建缓存
flutter clean

# 删除 Gradle 缓存
rm -rf ~/.gradle/caches/  # macOS/Linux
Remove-Item -Recurse -Force $env:USERPROFILE\.gradle\caches  # Windows

# 重新构建
flutter pub get
flutter build apk
```

### 4. 依赖下载失败

**症状**: `flutter pub get` 超时或失败

**解决**:
```bash
# 使用国内镜像
export PUB_HOSTED_URL=https://mirrors.tuna.tsinghua.edu.cn/dart-pub

# 或者使用阿里云镜像
export PUB_HOSTED_URL=https://pub.flutter-io.cn

# 重新获取依赖
flutter pub get
```

### 5. Android 许可问题

**症状**: `flutter doctor` 显示 Android 许可未接受

**解决**:
```bash
# 接受所有许可
flutter doctor --android-licenses

# 输入 'y' 接受所有许可
```

### 6. 构建时内存不足

**症状**: 构建过程中出现 OutOfMemoryError

**解决**:
```bash
# 增加 Gradle 内存分配
# 编辑 android/gradle.properties
org.gradle.jvmargs=-Xmx4096m
org.gradle.daemon=true
```
