# 分批实施计划（排除开发者平台）

## 批次划分

### 第一批：安全加固（Phase 1）- 1周
**目标**：修复安全漏洞，保障系统基础安全

| 序号 | 任务 | 工时 | 说明 |
|------|------|------|------|
| 1.1 | API Key 管理接口添加认证 | 0.5天 | 给 `api-key/admin/*` 添加 JwtAuthGuard + AdminGuard |
| 1.2 | 远程命令执行限制 | 0.5天 | 添加命令白名单，禁止危险命令 |
| 1.3 | Token 自动刷新机制 | 1天 | 后端验证 + 管理后台拦截器 + APP端逻辑 |
| 1.4 | 敏感操作审计日志 | 1天 | 创建 audit_logs 表，记录管理员操作 |
| - | **测试部署** | 1天 | 验证修复效果，部署上线 |
| - | **缓冲时间** | 1天 | 处理意外问题 |

**交付物**：
- 安全的 API Key 管理接口
- 受限的远程命令执行
- 自动刷新的 Token 机制
- 可查看的审计日志

---

### 第二批：API网关核心（Phase 2前半）- 1周
**目标**：实现用户可调用的 AI 服务接口

| 序号 | 任务 | 工时 | 说明 |
|------|------|------|------|
| 2.1 | Redis 限流系统 | 1天 | 部署 Redis，实现分布式限流 |
| 2.2 | AI 服务调用接口 | 2天 | 创建 AiController，实现完整调用链路 |
| 2.3 | 统一响应格式 | 0.5天 | 定义标准响应格式，实现拦截器 |
| - | **测试部署** | 1天 | 端到端测试 |
| - | **缓冲时间** | 0.5天 | - |

**交付物**：
- `POST /ai/chat` 接口（用户可调用）
- `POST /ai/transcribe` 接口（语音转写）
- Redis 限流保护
- 标准响应格式

---

### 第三批：智能路由与监控（Phase 2后半 + Phase 3前半）- 1周
**目标**：提升系统稳定性和可观测性

| 序号 | 任务 | 工时 | 说明 |
|------|------|------|------|
| 3.1 | API Key 健康检查 | 0.5天 | 定时检测 Key 可用性 |
| 3.2 | 智能路由与故障转移 | 1天 | 自动选择最优 Key，故障时切换 |
| 3.3 | 指标采集系统 | 1天 | 采集 QPS、延迟、错误率 |
| 3.4 | 基础监控仪表盘 | 1天 | 管理后台查看核心指标 |
| - | **测试部署** | 0.5天 | - |

**交付物**：
- 自动故障转移的 API 调用
- 管理后台监控页面（调用量、错误率、延迟）
