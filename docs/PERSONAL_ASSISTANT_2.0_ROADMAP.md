## 一、现有功能待完善点（推动 2.0 升级）

### 1.1 数据层能力缺口

| 缺口 | 现状 | 影响 | 2.0 方向 |
|------|------|------|---------|
| **无用户画像系统** | 知识图谱仅提取关键词，无用户偏好/习惯/目标记忆 | AI 分析每次都是"冷启动"，无法基于用户历史提供个性化建议 | 个人记忆系统 |
| **无对话上下文持久化** | AI 分析是单次请求，无多轮对话能力 | 用户无法追问、澄清，交互体验断裂 | 对话式助手 |
| **无向量化检索** | 全文搜索基于 FTS5，只能关键词匹配 | 无法语义搜索，找不到"意思相近但用词不同"的内容 | 知识库问答 RAG |
| **无时间线关联** | 记录之间只有关键词关联，无时间维度回溯 | 无法回答"我上周关于 XX 的思考"这类问题 | 时间线记忆 |

### 1.2 交互层能力缺口

| 缺口 | 现状 | 影响 | 2.0 方向 |
|------|------|------|---------|
| **纯被动触发** | AI 分析需用户手动点击，无主动推送 | 用户容易遗漏重要信息，助手感弱 | 主动助手 |
| **无任务闭环** | 智能待办工具独立存在，不与记录联动 | 会议记录里的 action items 无法自动转为任务 | 任务与日程管理 |
| **无语音唤醒** | 必须打开 App 才能使用语音功能 | 使用门槛高，无法做到"随时召唤" | 语音快捷助手 |
| **无今日概览** | 首页只有记录列表，无当日任务/提醒 | 用户需要主动翻找，效率低 | 今日卡片 |

### 1.3 输入层能力缺口

| 缺口 | 现状 | 影响 | 2.0 方向 |
|------|------|------|---------|
| **无视频转写** | 仅支持音频录音和 OCR | 无法处理视频课程/会议录像 | 多模态输入扩展 |
| **无网页剪藏** | 无法收藏外部文章/网页 | 知识库仅限于用户自己产生的内容 | 网页剪藏 |
| **无图片深度理解** | OCR 只提取文字，不理解图片内容 | 截图、图表、白板等内容无法分析 | 图片理解 |

### 1.4 自动化层能力缺口

| 缺口 | 现状 | 影响 | 2.0 方向 |
|------|------|------|---------|
| **工具各自独立** | 26 个工具无联动，需手动串联 | "录音→转写→提取任务→生成周报"需 4 步手动操作 | Agent 化工作流 |
| **无定时任务** | 所有操作需用户触发 | 无法实现"每天 18:00 总结今日记录" | 定时 Agent |
| **无外部集成** | 无法连接日历、邮件、Notion 等 | 数据孤岛，无法融入用户现有工作流 | MCP 外部集成 |

### 1.5 服务端能力缺口（参考 TASKS.md）

| 缺口 | 现状 | 影响 | 2.0 方向 |
|------|------|------|---------|
| **API 服务调用链路未完成** | TASKS.md 中 AI 服务调用接口待实现 | 云端 API Key 分发后无实际调用链路 | 完善服务端 AI 网关 |
| **无 Redis 限流** | 内存限流，重启后丢失 | 无法应对流量高峰 | Redis 限流系统 |
| **无监控告警** | 无 Prometheus/Grafana | 线上问题无法及时发现 | 监控仪表盘 |
| **支付系统未集成** | 微信支付/支付宝待接入 | 无法商业化变现 | 支付与订阅系统 |

### 方向 1：个人记忆系统

**核心能力**：让 AI 记住用户是谁、用户关心什么、用户的历史决策。

**具体功能**：
- 自动从记录中提取用户画像标签（职业、兴趣、目标、习惯）
- 维护用户偏好记忆（常用分析角色、输出格式偏好）
- 时间线记忆：按时间维度组织用户的思考轨迹
- 关联记忆：自动关联相关历史记录到当前分析

**技术要点**：
- 新增 `user_profiles` 表（Drift）
- AI 分析时自动注入用户画像到 system prompt
- 定期（每周）自动更新用户画像

### 方向 2：主动助手

**核心能力**：从"用户找功能"变为"助手找用户"。

**具体功能**：
- 每日晨间摘要：今日待办、今日日程、昨日未完成任务
- 智能提醒：基于记录内容识别 deadline，到期前主动提醒
- 关联推荐：查看记录时主动推荐相关历史记录
- 每周回顾：自动生成本周记录摘要和下周建议

**技术要点**：
- 本地通知系统（flutter_local_notifications）
- 后台任务调度（workmanager）
- AI 内容理解（自动识别时间、任务、deadline）

### 方向 3：知识库问答 RAG

**核心能力**：让用户用自然语言"问"自己的知识库。

**具体功能**：
- 全文向量化：所有记录内容 embedding 化
- 对话式查询：自然语言提问，AI 基于个人知识库回答
- 引用溯源：回答时标注来源记录
- 支持 `@记录标题` 引用特定记录

**技术要点**：
- 向量数据库选型：`sqlite-vec`（本地优先）或 `objectbox`
- Embedding API：复用现有 HttpClient 调用各提供商 embedding 接口
- RAG 流程：查询向量化 → 相似度检索 → 上下文组装 → LLM 生成

### 方向 4：任务与日程管理

**核心能力**：从记录中自动提取任务，形成闭环管理。

**具体功能**：
- 自动任务提取：AI 分析会议纪要/语音笔记，提取 action items
- 任务看板：状态流转（待办 → 进行中 → 已完成）
- 日历集成：与系统日历打通，任务同步到日历
- 循环任务：支持周期性提醒

**技术要点**：
- 新增 `tasks` 表（Drift）
- 日历权限申请与系统日历读写
- 工作台"智能待办"升级为"任务中心"

### 方向 5：多模态输入扩展

**核心能力**：不仅限于语音，支持更多输入形式。

**具体功能**：
- 视频转写：粘贴视频链接，提取音频转写
- 网页剪藏：一键收藏网页文章，自动摘要
- 图片深度理解：理解图表、截图、白板内容

**技术要点**：
- 视频下载/音频提取（依赖第三方库）
- 网页内容抓取 +  readability 解析
- 多模态模型调用（Gemini、Claude 等支持图片输入的模型）

### 方向 6：Agent 化工作流

**核心能力**：用户可配置自动化流程，减少重复操作。

**具体功能**：
- 自定义工作流：可视化配置"触发条件 → 动作"流程
- 预设模板：提供常用工作流模板（如"会议全流程"）
- 定时 Agent：设置定时执行的任务
- 外部集成：通过 MCP 协议连接外部服务

**技术要点**：
- 工作流引擎设计（触发器 → 条件判断 → 动作执行）
- MCP 客户端实现
- 工作流持久化与状态管理

## 三、实施路径建议

### Phase 1：基础升级（1-2 个月）

**目标**：建立个人助手的核心基础设施

| 功能 | 优先级 | 依赖 | 说明 |
|------|--------|------|------|
| 用户画像系统 | P0 | 无 | 从现有记录自动提取，新增表+AI分析流程 |
| 知识库问答 RAG | P0 | 向量存储 | 本地向量库 + embedding + 检索 |
| 每日摘要/提醒 | P1 | 用户画像 | 基于用户画像生成个性化摘要 |
| 任务自动提取 | P1 | 无 | 在现有 AI 分析中增加任务提取 prompt |

### Phase 2：能力深化（2-3 个月）

**目标**：强化助手主动性和交互体验

| 功能 | 优先级 | 依赖 | 说明 |
|------|--------|------|------|
| 任务看板 | P0 | 任务提取 | 将提取的任务可视化展示 |
| 语音快捷助手 | P1 | 无 | 应用内长按快捷按钮 + 语音交互 |
| 日历集成 | P1 | 任务看板 | 任务同步到系统日历 |
| 视频转写 | P2 | 无 | 新增输入方式 |

### Phase 3：生态扩展（3-6 个月）

**目标**：构建开放生态，融入用户工作流

| 功能 | 优先级 | 依赖 | 说明 |
|------|--------|------|------|
| 网页剪藏 | P1 | 无 | 外部知识输入 |
| Agent 工作流 | P2 | 无 | 自定义自动化 |
| MCP 外部集成 | P2 | Agent 工作流 | 连接日历、邮件、Notion 等 |
| 社区模板市场 | P3 | Agent 工作流 | 工作流模板分享 |

## 四、与现有版本的衔接

### 4.1 数据兼容性

- 现有 `records` 表结构不变，新增表通过 Drift migration 升级
- 用户画像从现有记录** retroactive 提取**（历史记录也能生成画像）
- 向量库**异步构建**（首次升级后后台逐步生成 embedding）

### 4.2 功能兼容性

- 所有现有功能保持不变
- 新增功能以"可选模块"形式提供
- 工作台工具可配置显示/隐藏

### 4.3 服务端衔接

- 优先完成 TASKS.md 中的 P0 任务（API 服务调用、Redis 限流）
- 支付系统完成后，可为 2.0 高级功能提供商业化支撑

## 五、关键决策记录

| 决策 | 选择 | 理由 |
|------|------|------|
| 向量数据库 | `sqlite-vec`（本地优先） | 符合畅记"本地优先、隐私保护"的定位 |
| 用户画像更新 | 每周自动 + 手动触发 | 平衡实时性与 API 成本 |
| RAG 实现 | 自研轻量方案 | 现有 HttpClient 已支持多模型，无需引入新依赖 |
| 语音唤醒 | 应用内优先 | 系统级集成复杂度高，先验证需求 |
| 工作流引擎 | 自研简单规则引擎 | 初期不需要复杂的工作流，规则触发即可 |

## 六、竞品差异化定位

| 竞品 | 定位 | 畅记云 2.0 差异化 |
|------|------|------------------|
| Muses | 卡片式笔记 + RAG 问答 | 更强的语音原生能力 + 多模型支持 |
| Get 笔记 | 语音速记 + 知识库 | 更深的 AI 分析（角色模板）+ 主动助手 |
| Memo AI | 视频/音频转写 | 更完整的个人助手闭环（任务+日程+提醒）|
| Notion AI | 文档协作 + AI | 语音原生 + 移动端优先 + 本地优先 |

**畅记云 2.0 核心差异化**：**"语音原生 + 本地优先 + 多模型 + 主动服务"** 的个人助手。
