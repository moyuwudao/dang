# 畅记 App - 账户/云服务/付费体系 完整方案

## CAPABILITY（能力定义）

**当前状态**：畅记是一款本地优先的 AI 语音笔记应用，用户必须自行配置 OpenAI/DeepSeek 等第三方 API Key 才能使用 AI 功能（语音转文字、AI 分析、工作台工具）。所有数据存储在本地 SQLite，支持 WebDAV 备份。

**目标状态**：建立完整的云端账户体系，用户无需配置任何 API Key 即可使用全部 AI 功能。通过云端统一调度 AI 资源，并建立充值付费体系实现商业化。

**核心能力**：
- 用户注册/登录/找回密码（手机号/邮箱/第三方登录）
- 云端数据同步与备份（替代/增强 WebDAV）
- 云端 AI 服务代理（用户无需配置 API Key）
- 充值付费系统（套餐/按量计费/订阅制）
- API 调用计量与配额管理
- 用户等级与权益体系

---

## CONSTRAINTS（约束条件）

### 业务规则
1. **零配置原则**：新用户注册后即可使用全部 AI 功能，无需配置任何 API Key
2. **向后兼容**：现有已配置自有 API Key 的用户可选择继续使用自有 Key 或切换到云服务
3. **数据主权**：用户数据加密存储，用户可随时导出/删除
4. **计费透明**：每次 API 调用消耗额度实时可见，余额不足时友好提示
5. **离线可用**：核心功能（录音、本地存储）在无网络时仍可正常使用

### 技术约束
- Flutter 客户端 + 后端服务架构（需新增后端）
- 数据库 schema 需支持用户 ID 关联
- 现有本地数据需支持迁移到云端账户
- 云端 AI 调用需做熔断、限流、降级
- 支付需符合国内法规（ICP 备案、支付牌照等）

### 安全约束
- 用户密码/手机号等敏感信息加密存储
- API Key 等密钥仅在服务端存储，绝不下发到客户端
- 通信全程 HTTPS + 证书固定
- 支付信息不落地，直接对接支付平台

| 模块 | 功能 | 优先级 |
|------|------|--------|
| 数据同步 | 录音记录、笔记、工具输出等全量同步 | P0 |

| 文件存储 | 音频文件、图片等云端存储 | P0 |

#### 2.2 同步策略
- **实时同步**：有网络时操作后立即同步
- **离线队列**：无网络时本地排队，恢复后批量同步
- **增量同步**：仅同步变更数据，减少流量
- **冲突处理**：时间戳优先 + 用户手动合并（复杂场景）

#### 2.3 云端 AI 代理架构
```

                                    v
                            [Auth Service] -- 验证 Token
                                    |
                                    v
                            [Quota Service] -- 检查额度
                                    |
                                    v
                            [AI Proxy Service] -- 调用 AI API
                                    |
                                    v
                            [计费 Service] -- 记录消耗
                                    |
                                    v
[客户端] <--(4.返回结果)---------- [Response]
```

#### 3.2 充值系统

### 四、API 付费使用系统（API Metering System）

#### 4.1 功能模块
| 模块 | 功能 | 优先级 |
|------|------|--------|
| 调用鉴权 | JWT Token 验证 + 权限检查 | P0 |
| 配额检查 | 调用前检查用户余额/套餐/订阅 | P0 |
| 用量计量 | 精确记录每次调用的消耗 | P0 |
| 额度控制 | 超限时拒绝或降级 | P0 |
| 用量统计 | 用户侧用量展示，运营侧统计 | P1 |
| 价格配置 | 运营后台动态调整价格 | P2 |

#### 4.2 调用流程
```
1. 客户端请求 → 携带 JWT Token
2. API Gateway 验证 Token 有效性
3. Quota Service 检查用户额度
   - 有订阅且未过期 → 允许（检查订阅配额）
   - 有套餐余额 → 允许（扣除套餐额度）
   - 有充值余额 → 允许（预扣或后扣）
   - 免费额度 → 允许（扣除免费额度）
   - 无额度 → 拒绝或降级
4. AI Proxy 调用实际 AI 服务
5. Metering Service 记录用量
6. Billing Service 扣费
7. 返回结果给客户端
```

#### 4.3 降级策略
- 余额不足时：提示充值，允许查看历史数据
- 免费版超额：提示升级，保留基础功能
- 服务端限流：排队或返回"服务繁忙"
- AI 服务异常：返回友好错误，不扣费

## IMPACT ASSESSMENT（影响评估）

### 正面影响
- **用户体验大幅提升**：零配置即可使用，降低使用门槛
- **商业化变现**：建立可持续的盈利模式
- **数据价值**：云端数据可支持更智能的 AI 功能
- **多设备体验**：无缝切换设备

### 负面影响
- **开发复杂度大增**：需要完整后端团队/能力
- **运营成本**：服务器、AI API、带宽等持续成本
- **合规风险**：需要处理备案、支付牌照、数据安全等
- **现有用户习惯改变**：已配置 API Key 的用户可能不适应

### 风险与缓解
| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| AI 成本过高 | 亏损 | 精确计量、动态定价、缓存策略 |
| 服务端宕机 | 全部用户不可用 | 多可用区部署、熔断降级 |
| 数据泄露 | 品牌/法律风险 | 加密存储、最小权限、审计日志 |
| 支付合规 | 无法收款 | 使用正规支付服务商、完成备案 |
| 用户不接受付费 | 转化率低 |  generous 免费额度、渐进式收费 |
