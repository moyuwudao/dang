# 畅记产品开发计划
## 面向"一人公司"的智能速记应用

---

## 📋 项目概述

### 产品定位
为"一人公司"用户打造的极速语音速记应用，支持BYOK（Bring Your Own Key）模式，通过OpenAI Whisper API实现语音转文字，配合OCR和AI增强功能，帮助创业者快速捕捉灵感和管理知识。

### 核心价值主张
- **3秒启动**：一键录音，极速捕获
- **零依赖后端**：纯客户端实现，数据本地存储
- **BYOK模式**：用户自带API Key，隐私安全
- **智能增强**：AI总结、OCR识别、标签管理

### 目标用户
- 一人公司创始人/自由职业者
- 产品经理/创业者
- 需要频繁记录灵感的知识工作者

## 🎯 开发里程碑

### 第一阶段（P0）：MVP核心 - 6周
**目标**：用户能在3秒内完成记录并成功获得转录文本

#### Week 1: 项目初始化 + 架构搭建
**交付物**：
- Flutter项目脚手架
- Riverpod + Drift + GoRouter 集成
- 基础主题和UI组件库
- 项目目录结构规范

**技术要点**：
```yaml
dependencies:
  flutter_riverpod: ^2.4.9
  drift: ^2.14.1
  drift_flutter: ^0.2.0
  go_router: ^13.0.0
  record: ^5.0.4
  openai_dart: ^0.3.0
  google_mlkit_text_recognition: ^0.11.0
  permission_handler: ^11.1.0
```

#### Week 2: BYOK配置 + 语音录制
**交付物**：
- API Key配置页面（验证+成本预估）
- 录音核心功能（波形显示+本地保存）
- Quick Settings Tile集成
- 录音权限管理

**关键功能**：
- API Key输入与验证
- Whisper模型选择（whisper-1）
- 成本预估计算器
- m4a格式录音
- 录音波形可视化
- 本地音频文件管理

#### Week 3: 转写流程 + 记录详情
**交付物**：
- Whisper API调用封装
- 文本展示页面
- 音频回放功能
- 编辑/删除/重试操作

**API调用流程**：
```
录音完成 → 上传音频 → Whisper API → 返回文本 → 保存到数据库
```

**错误处理**：
- API调用失败保留音频文件
- 网络错误自动重试
- API Key失效提示

#### Week 4: 记录列表 + 本地持久化
**交付物**：
- 时间线列表页面
- Drift数据库设计
- 离线队列机制
- 自动重试逻辑

**数据库设计**：
```sql
CREATE TABLE records (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  type TEXT NOT NULL, -- 'audio' or 'ocr'
  content TEXT,
  audio_path TEXT,
  created_at INTEGER NOT NULL,
  updated_at INTEGER NOT NULL,
  tags TEXT, -- JSON array
  transcription_status TEXT -- 'pending', 'success', 'failed'
);
```

#### Week 5: OCR速记 + 标签系统
**交付物**：
- 拍照/相册选择功能
- Google ML Kit OCR集成
- 简化版标签系统（1-3个自定义标签）
- 标签云快速筛选

**OCR流程**：
```
拍照/相册 → 图片预处理 → ML Kit识别 → 文本提取 → 保存记录
```

#### Week 6: 集成测试 + 打磨发布
**交付物**：
- 端到端测试用例
- UI/UX打磨
- 应用签名配置
- 内测分发包

**测试重点**：
- 3秒启动录音测试
- API调用成功率测试
- 离线场景测试
- 数据持久化测试

## 🛠️ 技术架构

### 技术栈选型

| 维度 | 技术选型 | 理由 |
|------|---------|------|
| 框架 | Flutter | 跨平台支持，未来可扩展iOS |
| 状态管理 | Riverpod | 轻量、编译时安全、适合中小项目 |
| 本地数据库 | Drift (SQLite) | Flutter生态最成熟ORM，内置FTS5 |
| 路由 | GoRouter | 声明式路由，深链接友好 |
| 本地OCR | Google ML Kit | 离线可用、免费、准确率够用 |
| 语音唤醒 | Quick Settings Tile | 无需特殊权限，用户体验好 |
| 录音格式 | m4a (AAC) | Whisper原生支持，体积小质量好 |

## ⚠️ 风险评估与应对

### 技术风险

| 风险 | 影响 | 概率 | 应对措施 |
|------|------|------|----------|
| Whisper API调用失败 | 高 | 中 | 实现离线队列、自动重试、错误提示 |
| 录音权限被拒绝 | 高 | 低 | 优雅降级、引导用户授权 |
| OCR识别准确率低 | 中 | 中 | 提供手动编辑、支持多张拍摄 |
| 数据库迁移问题 | 高 | 中 | 版本化数据库(schemaVersion)、onUpgrade迁移脚本、自动备份机制 |
| 覆盖安装数据丢失 | 高 | 中 | 数据库自动备份、手动备份入口、数据恢复功能 |

### 产品风险

| 风险 | 影响 | 概率 | 应对措施 |
|------|------|------|----------|
| 用户不愿提供API Key | 高 | 中 | 提供详细说明、安全承诺、未来自营API |
| 3秒启动体验不佳 | 高 | 中 | 多种启动方式、性能优化、用户测试 |
| 标签系统过于复杂 | 中 | 中 | 简化设计、提供预设标签、渐进式功能 |
| AI增强成本过高 | 中 | 低 | 成本预估提示、按需使用、缓存结果 |

### 市场风险

| 风险 | 影响 | 概率 | 应对措施 |
|------|------|------|----------|
| 竞品快速跟进 | 高 | 中 | 快速迭代、差异化功能、用户粘性 |
| 用户付费意愿低 | 高 | 中 | 免费基础版、增值服务、按需付费 |
| 平台政策变化 | 中 | 低 | 多平台支持、合规性审查、备选方案 |

## 🧪 用户验证策略

### MVP验证指标

**核心指标**：
- 5个内测用户中至少4个能在3秒内完成记录并成功获得转录文本
- API调用成功率 > 95%
- 用户日活留存率 > 40%（7日）

**次要指标**：
- 平均每条记录转录时间 < 30秒
- 用户满意度评分 > 4.0/5.0
- 功能使用率：录音 > 80%，OCR > 30%，标签 > 50%

### 用户测试计划

#### Week 5: 内测用户招募
- 目标：5-10名"一人公司"用户
- 渠道：创业社群、产品经理群、朋友圈
- 激励：免费使用、功能优先体验

#### Week 6: 用户测试执行
- 任务清单：
  1. 完成首次录音并查看转录结果
  2. 使用OCR识别一张图片
  3. 添加标签并筛选记录
  4. 编辑和删除记录
  5. 测试离线场景

- 数据收集：
  - 用户行为数据（埋点）
  - 录音和操作时长
  - 错误日志
  - 用户反馈问卷

#### Week 7: 结果分析与迭代
- 分析测试数据
- 整理用户反馈
